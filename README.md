# Web Portal Guide

本项目用于归档和发布多个独立 HTML 页面。这些页面本身不依赖任何特定域名或网站，仅作为通用前端资源进行收录与维护。

## 项目简介

本仓库主要存放一系列独立、自包含的 HTML 文件。每个页面均可单独打开、查看或嵌入。适合用于以下场景：

- 前端页面示例与样式归档
- 静态页面快速部署与分享
- 独立功能页面的版本管理

仓库本身不涉及任何具体网站或商业服务，仅作为页面资源的统一存放地。

## 目录说明

```
.
├── index.html          # 项目首页 / 导航页
├── pages/              # 存放独立 HTML 页面的目录
│   ├── example.html    # 示例页面
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── styles/
│   └── scripts/
└── README.md           # 本文件
```

- `pages/`：所有独立页面均归档于此，每个文件为一个完整页面。
- `assets/`：存放页面间共享的样式、脚本及静态资源。

## 页面归档说明

- 每个页面应为独立、可运行的 HTML 文件，尽量不依赖外部服务或动态接口。
- 页面之间无强耦合关系，可单独部署或引用。
- 页面命名建议使用英文小写单词，以连字符分隔。
- 如需引用公共资源，请统一存放在 `assets/` 目录下。

## 维护说明

- 本仓库为静态资源归档用途，不涉及后台或数据库。
- 欢迎提交改进或新增页面的 Pull Request，但请确保页面内容合规、无恶意代码。
- 提交前请检查页面是否可独立运行，并更新首页导航链接（如有）。
- 建议保持目录结构清晰，避免重复资源。
